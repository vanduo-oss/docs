Bubble / Popover

Basic Popover

<!-- Basic popover via data attribute -->
<button class="vd-btn vd-btn-primary"
        data-vd-bubble="This is a simple popover message.">
  Hover me
</button>

<!-- Alias: data-vd-popover also works -->
<button data-vd-popover="Same behaviour.">Alias</button>

Placement Variants

<button data-vd-bubble="Top"
        data-vd-bubble-placement="top">Top</button>

<button data-vd-bubble="Right"
        data-vd-bubble-placement="right">Right</button>

<button data-vd-bubble="Bottom"
        data-vd-bubble-placement="bottom">Bottom</button>

<button data-vd-bubble="Left"
        data-vd-bubble-placement="left">Left</button>

With Title & Close Button

<button data-vd-bubble="Body text for the popover."
        data-vd-bubble-title="Popover Title"
        data-vd-bubble-placement="bottom">
  Titled Popover
</button>

<!-- Generated markup -->
<div class="vd-bubble-content">
  <div class="vd-bubble-header">
    <strong>Popover Title</strong>
    <button class="vd-bubble-close" aria-label="Close">&times;</button>
  </div>
  <div class="vd-bubble-body">Body text</div>
</div>

API Reference

CSS Classes

Class Description Type
.vd-bubble-content Outer popover container (auto-generated by JS) Base
.vd-popover-content Alias for .vd-bubble-content Base
.vd-bubble-header Header section containing title and close button Structure
.vd-bubble-body Body section holding the popover text content Structure
.vd-bubble-close Close button within the header Action

Data Attributes

Attribute Description Default
data-vd-bubble Popover body text — placing this on any element makes it a trigger —
data-vd-popover Alias for data-vd-bubble —
data-vd-bubble-title Optional title shown in the popover header none
data-vd-bubble-placement Position relative to trigger: top, right, bottom, left top

JavaScript Methods

Method Description
VanduoBubble.init() Initialize all [data-vd-bubble] triggers on the page
VanduoBubble.show(trigger) Programmatically show the popover for the given trigger element
VanduoBubble.hide(trigger) Programmatically hide the popover for the given trigger element
VanduoBubble.hideAll() Dismiss every open popover on the page
VanduoBubble.destroy(trigger) Remove event listeners and generated DOM for the given trigger

Events

Event Description Detail
bubble:show Fired on the trigger element when its popover becomes visible { trigger, placement }
bubble:hide Fired on the trigger element when its popover is dismissed { trigger }

Accessibility

  • Trigger elements receive aria-describedby pointing to the popover ID
  • The popover uses role="tooltip" for untitled and role="dialog" for titled variants
  • Close buttons include aria-label="Close" for screen readers
  • Popover is dismissed on Escape keypress
  • Focus is trapped inside titled popovers and returned to the trigger on close
  • Popover auto-repositions to stay within the viewport (collision detection)
