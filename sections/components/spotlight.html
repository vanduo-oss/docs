Spotlight / Feature Discovery

Guide users through your interface with step-by-step tours. An overlay highlights target elements while a tooltip provides context and navigation controls.

Interactive Tour

Click the button below to start a 3-step feature discovery tour that highlights elements inside this demo area.

Search Bar
Notifications
Profile Menu

JavaScript API

const steps = [
  {
    target: '#search-input',
    title:  'Search',
    description: 'Find anything fast.'
  },
  {
    target: '.notifications-bell',
    title:  'Notifications',
    description: 'Your latest alerts.'
  }
];

// Start the tour
VanduoSpotlight.start(steps);

// Programmatic navigation
VanduoSpotlight.next();
VanduoSpotlight.prev();

// End the tour early
VanduoSpotlight.stop();

Declarative Usage

Attach a JSON array of steps directly to a trigger element with data-vd-spotlight. After Vanduo.init(), clicking that trigger starts the tour automatically.

<button
  data-vd-spotlight='[
    {"target":"#el-1","title":"First","description":"..."},
    {"target":"#el-2","title":"Second","description":"..."}
  ]'>
  Start Tour
</button>

API Reference

CSS Classes

Class Description
.vd-spotlight-overlay Full-screen semi-transparent backdrop with a cut-out around the target element.
.vd-spotlight-tooltip Floating card positioned near the highlighted target containing tour content.
.vd-spotlight-title Heading inside the tooltip.
.vd-spotlight-description Body text inside the tooltip explaining the highlighted feature.
.vd-spotlight-footer Bottom section of the tooltip holding navigation buttons and step counter.
.vd-spotlight-btn Base class for navigation buttons within the tooltip footer.
.vd-spotlight-btn-primary Primary-colored variant for the "Next" / "Finish" button.
.vd-spotlight-target Applied automatically to the currently highlighted DOM element. Raises it above the overlay.

Data Attributes

Attribute Description
data-vd-spotlight JSON array of step objects on a trigger element for declarative tour startup. Each step must include target; title and description are optional.

JavaScript Methods

Method Description
VanduoSpotlight.start(steps) Begins the tour. steps is an array of { target, title, description } objects. target is a CSS selector.
VanduoSpotlight.next() Advances to the next step. On the final step, the UI swaps the primary action to Done.
VanduoSpotlight.prev() Returns to the previous step. No-op if already on the first step.
VanduoSpotlight.stop() Ends the tour immediately, removes the overlay and tooltip, and restores target elements.

Events

Event Description
spotlight:step Fired on document when the tour moves to a new step. event.detail contains { index, step, total, data }.
spotlight:end Fired on document when the tour finishes or is stopped. event.detail contains { completedSteps, total, completed }.

Accessibility

  • The tooltip uses role="dialog" and aria-modal="true" to indicate a modal context to assistive technologies.
  • Pressing Escape or clicking the overlay ends the tour, and focus returns to the trigger when possible.
  • The overlay is marked aria-hidden="true" so screen readers skip the backdrop.
  • Each step's title and description are connected to the tooltip with aria-labelledby and aria-describedby.
  • The footer includes an aria-label announcing the current step count.
  • Keep step copy concise and make sure every highlighted target remains understandable even if a user dismisses the tour early.
