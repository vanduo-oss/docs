Toast Notifications

Transient popup notifications that appear and auto-dismiss after a set duration. Toasts support six screen positions, five semantic variants, action buttons, and persistent (non-auto-dismiss) mode. Use them for success confirmations, errors, warnings, and general updates.

JavaScript API

Programmatically show toasts with custom content, duration, and position.

// Simple toast
VanduoToast.show('Profile updated successfully', {
  variant: 'success',
  duration: 3000,
  position: 'top-right'
});

// Toast with action button
VanduoToast.show('New message received', {
  variant: 'primary',
  duration: 5000,
  position: 'bottom-left',
  actions: [
    { label: 'View', onClick: () => console.log('viewed') }
  ]
});

// Persistent toast (no auto-dismiss)
VanduoToast.show('Connection lost', {
  variant: 'error',
  duration: false,
  position: 'top-center'
});

// Dismiss all toasts
VanduoToast.dismissAll();

JavaScript Methods

MethodDescription
VanduoToast.show(message, options)Display a toast. options: { variant, duration, position, actions }. duration: false for persistent.
VanduoToast.dismiss(el)Dismiss a specific toast element immediately with exit animation.
VanduoToast.dismissAll()Remove all visible toasts at once.
VanduoToast.init()Wire up close buttons on static toast markup in the DOM.

Accessibility

  • Use role="alert" for error toasts so screen readers announce them immediately via the live region.
  • Use role="status" for success and info toasts so they are announced politely.
  • Each toast should have a close button with an aria-label="Dismiss notification" for keyboard users.
  • Toast containers use aria-live="polite" and aria-atomic="true" so additions are announced without interrupting the user.
  • Keep toast messages concise (under 120 characters) so they can be fully announced by screen readers.
  • Avoid using toasts for critical errors that require user action — use a modal or inline alert instead.
