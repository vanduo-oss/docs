Dropdown

Toggleable contextual overlays for displaying lists of links, actions, or custom content. Dropdowns support directional placement, glass styling, split buttons, and dark mode out of the box.

<!-- Basic dropdown -->
<div class="vd-dropdown">
  <button class="vd-btn vd-btn-primary vd-dropdown-toggle"
          data-vd-dropdown-toggle>
    Actions <i class="ph ph-caret-down"></i>
  </button>
  <ul class="vd-dropdown-menu">
    <li><a class="vd-dropdown-item" href="#">Edit</a></li>
    <li><a class="vd-dropdown-item" href="#">Duplicate</a></li>
    <li><div class="vd-dropdown-divider"></div></li>

  </ul>
</div>

API Reference

CSS Classes

ClassDescription
.vd-dropdownBase wrapper that establishes positioning context for the menu.
.vd-dropdown-toggleApplied to the trigger button. Adds caret icon and cursor behavior.
.vd-dropdown-menuThe popup list container. Hidden by default; shown when parent has .is-open.
.vd-dropdown-itemIndividual menu link or button.
.vd-dropdown-item.is-activeHighlighted state for the currently selected item.
.vd-dropdown-item.disabledMuted, non-interactive item.
.vd-dropdown-item-dangerRed-text variant for destructive actions (e.g., Delete).
.vd-dropdown-headerNon-interactive label inside the menu.
.vd-dropdown-dividerHorizontal separator between groups.
.vd-dropdown-splitModifier on .vd-dropdown for split-button layout.
.vd-dropdown-glassApplies glassmorphism background to the menu.
.vd-dropdown-dropupMenu opens above the trigger.
.vd-dropdown-droprightMenu opens to the right.
.vd-dropdown-dropleftMenu opens to the left.

Accessibility

  • The toggle button should have aria-haspopup="listbox" and toggle aria-expanded when the menu opens and closes.
  • Use <ul> and <li> for the menu to convey list semantics to screen readers.
  • Disabled items should use aria-disabled="true" on the anchor (or avoid using clickable elements).
  • Active items should have aria-current="true".
  • On open, focus should move to the first non-disabled item; on close, focus returns to the toggle button.
  • Escape closes the menu and returns focus to the toggle.
  • ↑ / ↓ arrow keys navigate items when the menu is open.
