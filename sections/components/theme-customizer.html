Theme Customizer

The Theme Customizer is a powerful component that allows users to personalize the framework's appearance in real-time. It provides controls for primary color, neutral color, border radius, font family, and color mode (light/dark/system).

Quick Start

Add a trigger button anywhere in your page to open the customizer panel:

<!-- Trigger button -->
<button class="vd-theme-customizer-trigger" 
        data-theme-customizer-trigger 
        aria-label="Open theme customizer">
    <i class="ph ph-paint-roller"></i>
</button>

<!-- Include the JS component -->
<script src="js/components/theme-customizer.js"></script>

Try it: Click the icon in the navbar to open the Theme Customizer.

Features

  • Primary Color: 18 color options from the Open Color palette
  • Neutral Color: 5 gray scale variants (Gray, Slate, Zinc, Neutral, Stone)
  • Border Radius: 5 presets (0, 0.125rem, 0.25rem, 0.375rem, 0.5rem)
  • Font Family: 10 bundled fonts including JetBrains Mono (default) plus Google Fonts collection
  • Color Mode: System (default), Dark, or Light

All preferences are persisted to localStorage and restored on page load.

JavaScript API

// Access the component
const customizer = Vanduo.components.themeCustomizer;

// Open/close programmatically
customizer.open();
customizer.close();
customizer.toggle();

// Get current state
const state = customizer.getState();
// Returns: { primary, neutral, radius, font, theme }

// Set values programmatically
customizer.applyPrimary('violet');
customizer.applyNeutral('slate');
customizer.applyRadius('0.375');
customizer.applyFont('inter');
customizer.applyTheme('dark');

// Or update multiple preferences at once
customizer.setPreferences({
    primary: 'violet',
    neutral: 'slate',
    radius: '0.375',
    font: 'inter',
    theme: 'dark'
});

// Reset to defaults
customizer.reset();

API Reference

Class Name / Attribute Description Type
.vd-theme-customizer-trigger Optional base styling component for the trigger button. Component
data-theme-customizer-trigger Attribute required on the trigger button to bind the click event that opens the panel. Attribute
data-primary Applied to <html>. Remaps --primary-* CSS variables to the selected color. State Attribute
data-neutral Applied to <html>. Remaps --gray-* CSS variables to the selected neutral scale. State Attribute
data-radius Applied to <html>. Sets --radius-scale for component border-radius. State Attribute
data-font Applied to <html>. Sets --font-family-base to the selected font stack. State Attribute
data-theme Applied to <html>. Forces light or dark mode. Removed when using system preference. State Attribute

Coordination with ThemeSwitcher

Auto-Coordination

When both ThemeCustomizer and ThemeSwitcher are present on the same page, they automatically stay in sync:

  • Changing theme via ThemeSwitcher button triggers primary color swap if using defaults
  • Changing theme via ThemeCustomizer panel updates ThemeSwitcher state
  • Preferences are shared via localStorage

<button class="vd-btn vd-btn-icon" data-toggle="theme">
    <i class="ph ph-moon">i>
button>


<button class="vd-theme-customizer-trigger" data-theme-customizer-trigger>
    <i class="ph ph-paint-roller">i>
button>

CSS Customization

Override the customizer panel styles:

/* Panel dimensions */
.theme-customizer-panel {
    --tc-panel-width: 320px;
    --tc-panel-max-height: 90vh;
}

/* Trigger button */
.theme-customizer-trigger {
    --tc-trigger-size: 40px;
    --tc-trigger-bg: var(--bg-secondary);
}

/* Color swatches */
.tc-color-swatch {
    --tc-swatch-size: 28px;
}

Events

Listen for customizer events:

// Panel opened
document.addEventListener('theme:panel-open', () => {
    console.log('Customizer opened');
});

// Panel closed
document.addEventListener('theme:panel-close', () => {
    console.log('Customizer closed');
});

// Any setting changed
document.addEventListener('theme:change', (e) => {
    console.log('Changed:', e.detail);
    // { type: 'primary-change', value: 'violet', state: { ... } }
});

// Settings reset
document.addEventListener('theme:reset', () => {
    console.log('Settings reset to defaults');
});
