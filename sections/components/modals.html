Modals

Overlay dialogs for confirmations, forms, and focused user tasks. Modals support four size tiers, glassmorphism styling, scrollable content, static backdrops, and are fully keyboard accessible.

Modal Sizes

Small Modal

This is the small modal size. Perfect for confirmations and short messages.

Default Modal

The default modal width is 377px — a comfortable size for most dialogs and forms.

Large Modal

Large modals (610px) work well for complex forms, image galleries, or multi-step wizards.

Extra Large Modal

Extra large modals (987px) span most of the viewport. Use sparingly for dashboards, preview panes, or large data tables.

<!-- Trigger button -->
<button class="vd-btn vd-btn-primary"
        data-modal="#my-modal">
  Open Modal
</button>

<!-- Modal markup -->
<div class="vd-modal" id="my-modal">
  <div class="vd-modal-dialog vd-modal-lg">
    <div class="vd-modal-content">

      <div class="vd-modal-header">
        <h5 class="vd-modal-title">Modal Title</h5>
        <button class="vd-modal-close"
                aria-label="Close">&times;</button>
      </div>

      <div class="vd-modal-body">
        <p>Modal content goes here.</p>
      </div>

      <div class="vd-modal-footer">
        <button class="vd-btn vd-btn-outline-secondary"
                data-dismiss="modal">Cancel</button>
        <button class="vd-btn vd-btn-primary">Save</button>
      </div>

    </div>
  </div>
</div>

Glass Modal

Glass Modal

Add .vd-modal-glass to the modal root for a frosted-glass backdrop and panel.

<!-- Glass modal -->
<div class="vd-modal vd-modal-glass" id="glass-modal">
  <div class="vd-modal-dialog">
    <div class="vd-modal-content">...</div>
  </div>
</div>

Static Backdrop

Confirm Delete

Clicking the backdrop won't close this modal. The user must explicitly choose an action.

<!-- Static backdrop (clicking outside won't close) -->
<div class="vd-modal vd-modal-static" id="confirm-modal">
  <div class="vd-modal-dialog vd-modal-sm">
    <div class="vd-modal-content">...</div>
  </div>
</div>

API Reference

CSS Classes

ClassDescription
.vd-modalBase modal container. Hidden by default; shown with .is-open.
.vd-modal-dialogCentered wrapper that constrains width and handles entrance animation.
.vd-modal-contentWhite/glass card containing header, body, and footer.
.vd-modal-headerTop bar with title and close button.
.vd-modal-titleHeading style for the modal title.
.vd-modal-closeClose button positioned at top-right of the header.
.vd-modal-bodyScrollable content area.
.vd-modal-footerBottom action bar, typically for buttons.
.vd-modal-smSmall width modifier on .vd-modal-dialog.
.vd-modal-lgLarge width modifier.
.vd-modal-xlExtra large width modifier.
.vd-modal-glassFrosted glass backdrop and translucent panel.
.vd-modal-staticBackdrop click does not close the modal.
.vd-modal-scrollableAllows the modal body to scroll independently when content overflows.
.is-openState class toggled by JavaScript to show the modal.

Data Attributes

AttributeDescription
data-modal="#id"Placed on a trigger button to open the targeted modal.
data-dismiss="modal"Placed on any button inside the modal to close it when clicked.

Accessibility

  • When opened, focus moves to the first focusable element inside the modal (typically the close button).
  • Focus is trapped within the modal while open — Tab cycles through modal contents only.
  • Escape closes the modal unless .vd-modal-static is present.
  • The modal container receives role="dialog" and aria-modal="true".
  • Use aria-labelledby on .vd-modal pointing to .vd-modal-title so screen readers announce the title.
  • On close, focus returns to the trigger element that opened the modal.
  • The backdrop blocks interaction with the rest of the page and is marked aria-hidden="true".
