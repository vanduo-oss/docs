Timeline

Display chronological events in a vertical track. By default the timeline is pure CSS — no JavaScript required. Supports alternating layout and color-coded markers for different event states.

Animated demos: Scroll to this section to watch each timeline reveal item-by-item. Add .vd-timeline-animated on .vd-timeline to opt in; the framework sets staggered --vd-timeline-reveal-delay per item (auto-init via Vanduo.init()). With .vd-timeline-alternating, items animate in from the left and right on desktop (same stagger); on narrow viewports the motion stays vertical. Add .vd-timeline-playback together with .vd-timeline-animated to drive reveals from buttons (data-vd-timeline-prev, data-vd-timeline-next, data-vd-timeline-play, data-vd-timeline-pause) in the direct parent of the timeline (e.g. .vd-card-body wrapping both the toolbar and the list)—scroll-based IntersectionObserver is not used in playback mode.

API Reference

CSS Classes

Class Description
.vd-timeline Base container. Draws a vertical line and positions items along it.
.vd-timeline-item Individual event wrapper containing a marker and content block.
.vd-timeline-marker Circular dot positioned on the timeline track. Default size is 12px.
.vd-timeline-marker-lg Larger marker (24px) that can contain an icon.
.vd-timeline-content Text block beside the marker holding title, date, and description.
.vd-timeline-title Heading text for the event.
.vd-timeline-date Small muted text for the event date or timestamp.
.vd-timeline-text Body text describing the event in detail.
.vd-timeline-alternating Modifier that alternates items left and right of the center line (desktop only).
.vd-timeline-animated Optional modifier (use with .vd-timeline) for fade/slide-in and marker pop-in. With scroll (default), items gain .is-revealed when scrolled into view. With .vd-timeline-playback, items are revealed only via controls (see below). Delays use --vd-timeline-reveal-delay (set per item by VanduoTimeline).
.vd-timeline-playback Use with .vd-timeline-animated. Disables scroll-driven reveal; place data-vd-timeline-prev, data-vd-timeline-next, data-vd-timeline-play, and data-vd-timeline-pause on buttons inside the timeline’s direct parent (e.g. toolbar and list share a .vd-card-body). Cumulative reveal: next/back add or remove .is-revealed on items in order; play auto-advances on an interval until the end or pause.
--vd-timeline-reveal-delay Per-item transition delay for the reveal animation (inline-set when using the animated modifier; first items stagger ~140ms apart, capped after the eighth).
.vd-timeline-success Colors the marker with the success (green) palette.
.vd-timeline-warning Colors the marker with the warning (amber) palette.
.vd-timeline-danger Colors the marker with the danger (red) palette.
Default: CSS only. Static timelines need no JavaScript. With .vd-timeline-animated alone, the framework runs one shared IntersectionObserver (auto-init via Vanduo.init()) to add .is-revealed. With .vd-timeline-playback, there is no observer—only the buttons. Use VanduoTimeline.reinit() after injecting new timeline markup.

Accessibility

  • Use an <ol> or add role="list" on the container and role="listitem" on each .vd-timeline-item so events are announced as an ordered sequence.
  • Markers are decorative — apply aria-hidden="true" on .vd-timeline-marker icons.
  • Ensure .vd-timeline-date text uses a <time> element with a datetime attribute for machine-readable dates.
  • Color variants must not be the sole indicator of state — pair them with text labels or icons.
  • The alternating layout collapses to a single column on narrow viewports, maintaining natural reading order.
