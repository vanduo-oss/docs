Lazy Load Utility

The VanduoLazyLoad utility uses the browser's IntersectionObserver to defer loading content until it's needed — when the target element scrolls into view. It works at two levels: a low-level observer API for any use-case, and a high-level section-fetcher that replaces an element with a fetched HTML partial.

Faster Initial Load

Only the visible viewport is fetched at startup. Off-screen content waits.

Zero Config

Add data-vd-lazy to any element and call Vanduo.init(). Done.

Full JS API

Use VanduoLazyLoad.observe() and loadSection() for complete control.

Attribute-driven Usage

The simplest way — add data-vd-lazy with the URL to fetch, then call Vanduo.init(). When the element scrolls into view, a placeholder appears while the content fetches, then the element is replaced with the loaded HTML.

<div
  data-vd-lazy="./sections/my-section.html"
  data-vd-lazy-placeholder="skeleton">
</div>

<!-- Vanduo.init() wires all [data-vd-lazy] elements automatically -->
<script>Vanduo.init();</script>

Placeholder Options

Value Result
"skeleton" (default) Animated skeleton card with shimmer and four-color spinner overlay
"spinner" Centered four-color spinner, minimal height
Any HTML string Your custom HTML rendered while loading

VanduoLazyLoad.loadSection(url, el, options?)

High-level API: fetches an HTML partial and injects it into el when the element enters the viewport. Shows a placeholder immediately.

const container = document.getElementById('my-section');

VanduoLazyLoad.loadSection('./sections/heavy-content.html', container, {
  placeholder: 'skeleton',    // 'skeleton' | 'spinner' | HTMLString
  threshold:   0,            // IntersectionObserver threshold (0–1)
  rootMargin:  '0px',        // IntersectionObserver rootMargin
  onLoaded:    (el) => { console.log('loaded!', el); },
  onError:     (err) => { console.error(err); }
});

Events

Event Fired on When
lazysection:loading containerEl Immediately after placeholder is shown
lazysection:loaded containerEl After content is injected successfully
lazysection:error containerEl After a fetch failure (404, network error, etc.)
Security note: loadSection() rejects any URL that resolves to a different origin. Only same-origin and relative URLs are allowed. Cross-origin requests are blocked silently with a console error.

Low-level Observer API

For full control, use the low-level API. The callback fires once when the element enters the viewport, then the element is automatically unobserved.

const el = document.querySelector('#my-element');

// Observe — callback fires once on first intersection, then auto-unobserves
VanduoLazyLoad.observe(el, (target) => {
  target.classList.add('is-visible');
  // load an image, animate in, fetch data...
}, { threshold: 0.2, rootMargin: '-50px' });

// Cancel a single element
VanduoLazyLoad.unobserve(el);

// Cancel all active observations
VanduoLazyLoad.unobserveAll();

API Reference

Method Parameters Description
