Floating Action Button (FAB)

Basic FAB

The circular FAB is designed for the single most important action on a screen. Use an icon that clearly represents the action.

Tip: Add .vd-fab-fixed to pin the FAB to the viewport corner. Without it, the FAB flows inline like a regular button.

<!-- Inline FAB (no fixed positioning) -->
<button class="vd-fab" aria-label="Add">
  <i class="ph ph-plus"></i>
</button>

<!-- Fixed to bottom-right of viewport -->
<button class="vd-fab vd-fab-fixed" aria-label="Add">
  <i class="ph ph-plus"></i>
</button>

Extended FAB (with Label)

When the icon alone isn't enough, use the extended variant to pair an icon with a text label. The pill shape keeps the elevated FAB feel while improving clarity.

<button class="vd-fab vd-fab-extended">
  <i class="ph ph-plus"></i>Create
</button>

<button class="vd-fab vd-fab-extended vd-fab-success">
  <i class="ph ph-check"></i>Approve
</button>

Speed Dial Menu

Group related actions behind a single FAB trigger. Clicking the main button reveals a stack of smaller action buttons with a staggered entrance animation.

Click the FAB above to toggle the speed dial. Uses a simple .is-open class toggle and aria-expanded — no JS framework required.

<!-- Speed dial FAB menu -->
<div class="vd-fab-menu">
  <button class="vd-fab" aria-label="Toggle menu"
          aria-expanded="false"
          onclick="this.parentElement.classList.toggle('is-open')">
    <i class="ph ph-plus"></i>
  </button>
  <div class="vd-fab-actions">
    <button class="vd-fab vd-fab-sm" aria-label="Upload">...</button>
    <button class="vd-fab vd-fab-sm" aria-label="Photo">...</button>
  </div>
</div>

<!-- Fixed speed dial in bottom-right corner -->
<div class="vd-fab-menu vd-fab-fixed">...</div>

When to Use a FAB

Do

  • Use for the single most important action on a screen (e.g. "New message", "Add item")
  • Use .vd-fab-fixed or a position class to anchor it to the viewport
  • Keep icon-only FABs to universally understood icons (plus, edit, share)
  • Use the extended variant when the icon alone may be ambiguous
  • Use the speed dial pattern to group 3–6 related actions

Don't

  • Don't place more than one fixed FAB on a page — it dilutes focus
  • Don't use a FAB for minor or infrequent actions — use a regular .vd-btn
  • Don't skip aria-label on icon-only FABs
  • Don't use more than 6 actions in a speed dial — it becomes overwhelming

JavaScript

The FAB is a CSS-only component. Speed dial toggling uses a simple .is-open class toggle — no dedicated JS module is required. You can add your own click handler or use the inline onclick example above.

Accessibility

  • Icon-only FABs require aria-label describing the action
  • Extended FABs with visible text labels do not need aria-label
  • Speed dial trigger should use aria-expanded reflecting menu state
  • Speed dial action container uses role="menu" and children use role="menuitem"
  • Focus order follows the DOM: trigger first, then actions from top to bottom
  • FABs support standard :focus-visible outline for keyboard users
