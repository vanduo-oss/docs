Speed Dial Menu

Click the FAB to toggle the speed dial actions. Uses a simple .is-open class toggle — no JS framework required.

<!-- Speed dial FAB menu -->
<div class="vd-fab-menu">
  <button class="vd-fab" aria-label="Toggle menu"
          onclick="this.parentElement.classList.toggle('is-open')">
    <i class="ph ph-plus"></i>
  </button>
  <div class="vd-fab-action">
    <button class="vd-fab vd-fab-sm">...</button>
    <button class="vd-fab vd-fab-sm">...</button>
  </div>
</div>

CSS Classes

Class Description Type

.vd-fab-sm Small FAB (40×40px) Size
.vd-fab-lg Large FAB (72×72px) Size
.vd-fab-extended Pill-shaped FAB with icon + text label Variant
.vd-fab-bottom-left Fixed position: bottom-left corner of the viewport Position
.vd-fab-top-right Fixed position: top-right corner of the viewport Position
.vd-fab-success Success/green colour variant Color
.vd-fab-danger Danger/red colour variant Color
.vd-fab-menu Container for a speed-dial group of FABs Structure
.vd-fab-action Wrapper for the child action buttons inside a speed dial Structure
.is-open Toggle class on .vd-fab-menu to reveal speed dial actions State

JavaScript

The FAB is a CSS-only component. Speed dial toggling uses a simple .is-open class toggle — no dedicated JS module is required. You can add your own click handler or use the inline onclick example above.

Accessibility

  • Icon-only FABs require aria-label describing the action
  • Extended FABs with visible text labels do not need aria-label
  • Speed dial trigger should use aria-expanded reflecting menu state
  • Speed dial action container uses role="menu" and children use role="menuitem"
  • Focus order follows the DOM: trigger first, then actions from top to bottom
  • FABs support standard :focus-visible outline for keyboard users
