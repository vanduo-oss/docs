Global Lazy Loading Guide

This guide walks you through using VanduoLazyLoad to build pages that load content on demand. Instead of shipping all HTML upfront, sections are fetched and injected only when they scroll into view — giving users a fast first paint and a smooth reading experience.

Why Lazy Loading?

A documentation site can have hundreds of kilobytes of HTML across dozens of sections. Sending it all on the first request is wasteful — most users never scroll to the bottom. Lazy loading solves this by splitting content into small partials that are fetched on demand.

Faster First Paint

Ship only what's visible. The browser renders the page while the rest is pending.

Less Bandwidth

Users who don't scroll never download off-screen content.

Composable Sections

Each section is an independent HTML partial — easy to author and maintain separately.

Step 1 Include VanduoLazyLoad

VanduoLazyLoad is part of the Vanduo Framework and is included automatically in the standard bundle. If you're loading scripts individually, include it after the main framework files:

<!-- Option A: Full IIFE bundle (already includes lazy-load) -->
<script src="vanduo.min.js"></script>

<!-- Option B: Individual scripts -->
<script src="js/utils/helpers.js"></script>
<script src="js/vanduo.js"></script>
<script src="js/components/lazy-load.js"></script>

Step 2 Create HTML Partials

Each lazy-loaded section is a standalone HTML fragment — just the content, no <html> or <body> wrapper. The root element should have a matching id so scroll-spy and anchor linking work correctly.

<section id="intro">
  <h2>Introduction</h2>
  <p>This content was loaded lazily.</p>
</section>

Step 3 The Simplest Usage: HTML Attributes

Add data-vd-lazy pointing to your partial, and optionally data-vd-lazy-placeholder for the loading state. Call Vanduo.init() once and everything is wired automatically.

<body>

  <!-- This section loads immediately (it's visible on page load) -->
  <div
    data-vd-lazy="./sections/intro.html"
    data-vd-lazy-placeholder="skeleton">
  </div>

  <!-- Below-fold section — fetched when user scrolls here -->
  <div
    data-vd-lazy="./sections/details.html"
    data-vd-lazy-placeholder="spinner">
  </div>

  <script>Vanduo.init();</script>

</body>
How it works: Vanduo.init() calls VanduoLazyLoad.init(), which scans the DOM for [data-vd-lazy] elements, shows the placeholder immediately, then uses IntersectionObserver to trigger the fetch when the element enters the viewport.

Step 4 Programmatic Usage

For dynamic UIs where elements are created at runtime, use the JS API directly:

// High-level: section fetcher
const el = document.getElementById('section-container');

VanduoLazyLoad.loadSection('./sections/heavy.html', el, {
  placeholder: 'skeleton',
  onLoaded: (container) => {
    console.log('Section ready!', container);
  },
  onError: (err) => {
    console.error('Could not load section:', err.message);
  }
});

// Low-level: generic observe
const img = document.querySelector('img[data-src]');

VanduoLazyLoad.observe(img, (target) => {
  target.src = target.dataset.src;  // swap src when visible
}, { threshold: 0.1 });

Listening to Events

All events bubble from the container element, so you can listen globally or on the target:

// Global listeners (bubble from any lazy container)
document.addEventListener('lazysection:loading', (e) => {
  console.log('Fetching:', e.detail.url);
});

document.addEventListener('lazysection:loaded', (e) => {
  console.log('Loaded:', e.detail.url, 'into', e.target);
  // Re-run analytics, highlight code, etc.
});

document.addEventListener('lazysection:error', (e) => {
  console.error('Failed:', e.detail.url, e.detail.error);
  // Show a retry button, report to Sentry, etc.
});

Real-world Patterns

Lazy Images

Use observe() to swap data-src → src when images enter view — native browser lazy loading without the loading="lazy" attribute.

Lazy Charts

Initialize chart libraries only when the chart container scrolls into view — avoids unnecessary JS execution on content users never see.

Paginated Content

Place a sentinel element at the bottom of a list. When it enters view, fetch the next page and append it — infinite scroll without a framework.

Scroll Animations

Use observe() to add an is-visible class when elements enter view, triggering CSS transitions for fade-in, slide-up, or reveal effects.
