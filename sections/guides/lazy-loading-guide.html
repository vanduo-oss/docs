Global Lazy Loading Guide

Use VanduoLazyLoad to fetch same-origin HTML, hydrate injected content, and observe expensive work only when it is needed. In v1.4.1 the important architecture change is scoped hydration: loaded content calls Vanduo.init(container) instead of reinitializing the entire page.

Step 1 Attribute-Driven Sections

Add data-vd-lazy to an empty container. Vanduo fetches the partial when the container enters the viewport.

<div
  data-vd-lazy="./sections/pricing.html"
  data-vd-lazy-placeholder="skeleton">
</div>

<script src="dist/vanduo.min.js"></script>
<script>Vanduo.init();</script>
After the partial is injected, v1.4.1 initializes only that lazy container. Existing dropdowns, tabs, tooltips, and other components outside the container are not touched.

Programmatic Section Loading

const container = document.querySelector('#pricing-slot');

VanduoLazyLoad.loadSection('./sections/pricing.html', container, {
  placeholder: 'skeleton',
  onLoaded: (root) => {
    console.log('Loaded and initialized:', root);
  },
  onError: (error) => {
    console.error(error.message);
  }
});

The canonical registry name is lazyLoad, but the convenience global VanduoLazyLoad remains available in v1.4.x.

Generic Observation

document.querySelectorAll('img[data-src]').forEach((img) => {
  VanduoLazyLoad.observe(img, (el) => {
    el.src = el.dataset.src;
    el.removeAttribute('data-src');
  }, { rootMargin: '200px' });
});

Observers are registered with the v1.4 lifecycle manager, so scoped destroy calls can clean them up.

API Reference

Method Description
observe(el, cb, opts?) Runs cb once when el enters view, then unobserves it.
unobserve(el) Stops observing one element.
unobserveAll() Stops all lazy-load observations.
loadSection(url, el, opts?) Fetches same-origin HTML into el, emits lazy-section events, and initializes only el.
init(root?) Wires [data-vd-lazy] containers inside root or the full document.
loadSection() only accepts same-origin or relative URLs. Fetched HTML is convenient for trusted partials; do not use it as a sanitization boundary for untrusted CMS/user content.
