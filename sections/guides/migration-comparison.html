Migration Comparison Guide

Switching frameworks? This guide translates familiar patterns from Bootstrap and Tailwind into Vanduo's component-first, zero-build approach. Side-by-side code shows the same UI in each framework so you can map your mental model.

v1.4.1 migration compass: New integration code should use scoped Vanduo.init(root) / Vanduo.destroy(root), canonical lowerCamelCase registry names such as lazyLoad, and --vd-* semantic tokens for theme overrides. Legacy unprefixed token aliases are no longer shipped.

Coming from Bootstrap?

Bootstrap and Vanduo are both component-first — you compose pre-built components rather than stacking utility classes. The main shift is Vanduo's vd- prefix and Fibonacci spacing. Here's the mapping:

Bootstrap Class Vanduo Equivalent Notes
.container.vd-containerSame concept — centered, max-width wrapper
.container-fluid.vd-container-fluidFull-width variant
.row.vd-rowHorizontal flex container for columns
.col-md-6.vd-col-md-6Same 12-column grid, same breakpoints
.btn .btn-primary.vd-btn .vd-btn-primaryAdd vd- prefix to both base and variant
.card.vd-cardCard structure is nearly identical
.alert .alert-success.vd-alert .vd-alert-successSame color variants
.navbar.vd-navbarResponsive hamburger built in
.form-control.vd-inputInside .vd-form-group wrapper
.badge.vd-badgeSame sizing and color variants
.table.vd-tableAdd .vd-table-striped for zebra
.modal.vd-modalTrigger with data-modal="#id"
.dropdown.vd-dropdownSimilar markup structure
.d-flex.vd-d-flexDisplay utilities carry the vd- prefix
.justify-content-center.justify-centerFlexbox utilities drop vd- for brevity
.text-center.vd-text-centerText alignment keeps the prefix
.mt-3 (16px).vd-mt-5 (55px)Fibonacci scale: 1=8px, 2=13px, 3=21px, 4=34px, 5=55px

Side-by-Side: Login Card

The same login card built in Bootstrap 5 and Vanduo:

<!-- Bootstrap -->
<div class="card" style="width: 18rem;">
  <div class="card-body">
    <h5 class="card-title">Sign In</h5>
    <div class="mb-3">
      <label class="form-label">Email</label>
      <input type="email"
             class="form-control">
    </div>
    <button class="btn btn-primary w-100">
      Sign In
    </button>
  </div>
</div>
<!-- Vanduo -->
<div class="vd-card" style="max-width: 18rem;">
  <div class="vd-card-body">
    <h5>Sign In</h5>
    <div class="vd-form-group">
      <label>Email</label>
      <input type="email"
             class="vd-input">
    </div>
    <button class="vd-btn vd-btn-primary vd-btn-block">
      Sign In
    </button>
  </div>
</div>
The pattern: Add vd- to almost every class. The exceptions are flexbox alignment utilities (justify-*, items-*) which drop the prefix for brevity. Spacing moves from Bootstrap's 4px base to Fibonacci numbers.

Coming from Tailwind CSS?

Tailwind is utility-first — you compose 15-30 utility classes per element. Vanduo is component-first — one class gives you a complete, styled component. The mental model shift is from "build everything from atoms" to "assemble from Lego bricks."

Conceptual Translation

Concept Tailwind Approach Vanduo Approach
Philosophy Utility-first: 20+ classes per element Component-first: 1-3 classes per element
Spacing m-4 = 16px (4×4px base) vd-mb-5 = 55px (Fibonacci scale)
Colors bg-blue-500, text-slate-700 vd-btn-primary (semantic) or vd-bg-blue-5 (per-hue)
Dark mode dark:bg-gray-800 on every element data-theme="dark" on <html> — everything adapts
Build step Required (JIT compiler, purge) Optional — CDN link works out of the box
Customization tailwind.config.js — build-time CSS variables — runtime, no rebuild
Card bg-white rounded-lg shadow-md p-6 vd-card (includes all of the above)
Button bg-blue-600 text-white px-4 py-2 rounded vd-btn vd-btn-primary

Side-by-Side: Alert Banner

A success alert in Tailwind and Vanduo:

<!-- Tailwind — 8 utility classes -->
<div class="bg-green-100 border border-green-400 text-green-700 px-4 py-3 rounded relative" role="alert">
  <strong class="font-bold">Success!</strong>
  <span class="block sm:inline">Your profile has been updated.</span>
</div>
<!-- Vanduo — 2 component classes -->
<div class="vd-alert vd-alert-success" role="alert">
  <strong>Success!</strong>
  Your profile has been updated.
</div>
The trade-off: Tailwind gives you pixel-perfect control with more classes. Vanduo gives you consistency and speed with fewer classes. Both can coexist — you can use Tailwind for custom layouts and Vanduo for standard components.

Key Differences at a Glance

Component-First

One class gives you a complete, styled, accessible component. Not 20 utility classes.

Fibonacci Spacing

Spacing follows the golden ratio: 8, 13, 21, 34, 55px. Not an arbitrary 4px grid.

Built-In Dark Mode

Flip data-theme="dark" on <html>. Every component adapts automatically. No manual dark variants.

Runtime Theming

Change colors, fonts, and radius via CSS variables in the browser. No rebuild or purge step.

Zero Build Tools

Drop a CDN link into any HTML file and start building. No npm install, no config file, no compiler.

Semantic HTML

Components use proper elements: <button> for actions, <nav> for navigation, <table> for data.

Why Switch to Vanduo?

Three reasons teams migrate:

  • Smaller bundle. Vanduo's CSS is a single file (~45KB gzipped). No purge step, no tree-shaking config, no unused class scanning.
  • Faster to prototype. Copy a starter template and ship a landing page in 30 minutes. No config file to tweak.
  • No JS framework lock-in. Vanduo works with plain HTML, React, Vue, Svelte, or server-rendered templates. One framework, any stack.
Not sure? You don't have to migrate everything at once. Use Vanduo for new pages while keeping your existing framework on legacy pages. Both can coexist in the same project.
