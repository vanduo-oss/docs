CSS Variables & Theming Guide

Vanduo v1.4.0 uses CSS custom properties as its public theme API. The important change is simple: author new themes and examples with canonical --vd-* semantic tokens. Older unprefixed semantic tokens still work through the 1.4.x line, but they are now compatibility aliases.

Token Tiers

Tier Examples Use
Palette primitives --primary-5, --gray-8, --success-6 Raw implementation scales. Use when composing framework tokens, not as your first override target.
Canonical semantic API --vd-color-primary, --vd-bg-primary, --vd-text-secondary The preferred public API for application themes and component examples.
Compatibility aliases --color-primary, --bg-primary, --text-secondary Supported in 1.4.x for existing sites. New docs and app code should use --vd-*.

Brand Override

Load your CSS after Vanduo and override the canonical semantic tokens you want to change.

:root {
  --vd-color-primary: #6d28d9;
  --vd-color-primary-hover: #5b21b6;
  --vd-color-primary-active: #4c1d95;
  --vd-color-primary-rgb: 109, 40, 217;

  --vd-bg-primary: #ffffff;
  --vd-bg-secondary: #f8fafc;
  --vd-text-primary: #111827;
  --vd-text-secondary: #475569;
  --vd-border-color: #dbe3ee;
}

Dark Mode

Dark mode is just another semantic-token assignment on html[data-theme="dark"].

html[data-theme="dark"] {
  --vd-bg-primary: #0f172a;
  --vd-bg-secondary: #1e293b;
  --vd-text-primary: #f8fafc;
  --vd-text-secondary: #cbd5e1;
  --vd-border-color: #334155;
}

Components consume semantic tokens, so they inherit the active theme without dark-mode-specific component selectors.

Component Tokens

Component-level tokens still tune individual components. Prefer canonical semantic values inside those overrides.

:root {
  --btn-border-radius: 9999px;
  --card-border-radius: 0.75rem;
  --input-border-radius: 0.5rem;
}

.billing-card {
  --card-bg: var(--vd-bg-secondary);
  --card-border-color: var(--vd-border-color);
}

Migrating Existing Themes

Existing 1.3.x themes do not need to break. Vanduo maps legacy semantic aliases back to canonical tokens in 1.4.x. For new work, move aliases over gradually:

/* 1.3.x style: still supported in 1.4.x */
:root {
  --color-primary: #2563eb;
  --bg-primary: #ffffff;
}

/* 1.4.0 style: preferred for new code */
:root {
  --vd-color-primary: #2563eb;
  --vd-bg-primary: #ffffff;
}
Raw palette scales such as --primary-5 and --gray-8 are intentionally unchanged in v1.4.0. They remain implementation primitives behind the semantic API.
