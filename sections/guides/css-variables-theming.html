CSS Variables & Theming Guide

Vanduo v1.4.1 makes --vd-* the strict public CSS custom property API. Author new themes, examples, and component overrides with canonical --vd-* tokens only; unprefixed token aliases are no longer shipped.

Token Tiers

Tier Examples Use
Palette primitives --vd-primary-5, --vd-gray-8, --vd-success-6 Raw implementation scales. Use when composing framework tokens, not as your first override target.
Canonical semantic API --vd-color-primary, --vd-bg-primary, --vd-text-secondary The preferred public API for application themes and component examples.
Component tokens --vd-card-bg, --vd-btn-border-radius, --vd-input-border-color Fine-grained component hooks. Keep these in the same namespace as semantic tokens.

Brand Override

Load your CSS after Vanduo and override the canonical semantic tokens you want to change.

:root {
  --vd-color-primary: #6d28d9;
  --vd-color-primary-hover: #5b21b6;
  --vd-color-primary-active: #4c1d95;
  --vd-color-primary-rgb: 109, 40, 217;

  --vd-bg-primary: #ffffff;
  --vd-bg-secondary: #f8fafc;
  --vd-text-primary: #111827;
  --vd-text-secondary: #475569;
  --vd-border-color: #dbe3ee;
}

Dark Mode

Dark mode is just another semantic-token assignment on html[data-theme="dark"].

html[data-theme="dark"] {
  --vd-bg-primary: #0f172a;
  --vd-bg-secondary: #1e293b;
  --vd-text-primary: #f8fafc;
  --vd-text-secondary: #cbd5e1;
  --vd-border-color: #334155;
}

Components consume semantic tokens, so they inherit the active theme without dark-mode-specific component selectors.

Component Tokens

Component-level tokens still tune individual components. Prefer canonical semantic values inside those overrides.

:root {
  --vd-btn-border-radius: 9999px;
  --vd-card-border-radius: 0.75rem;
  --vd-input-border-radius: 0.5rem;
}

.billing-card {
  --vd-card-bg: var(--vd-bg-secondary);
  --vd-card-border-color: var(--vd-border-color);
}

Migrating Existing Themes

Existing 1.3.x and 1.4.0 themes should migrate unprefixed token overrides before adopting v1.4.1. The shipped framework no longer maps legacy semantic aliases back to canonical tokens.

/* v1.4.1 strict token namespace */
:root {
  --vd-color-primary: #2563eb;
  --vd-bg-primary: #ffffff;
}
Raw palette scales such as --vd-primary-5 and --vd-gray-8 also live in the strict namespace. They remain implementation primitives behind the semantic API.
