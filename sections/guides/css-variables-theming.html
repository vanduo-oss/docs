CSS Variables & Theming Guide

Vanduo is built on top of a powerful CSS variables (Custom Properties) foundation. This means you can fundamentally alter the look and feel of the entire framework just by changing a few variables in your root CSS file, without ever needing a preprocessor like Sass.

The Three Variable Tiers

Vanduo's variables are layered to give you massive control with minimal effort:

  1. 1. Theme Colors: The highest level. Defines what "Primary" or "Success" actually means.
  2. 2. Component Tokens: Variables scoped to specific components (e.g., --btn-height or --card-bg).
  3. 3. Utility Variables: Core infrastructure like spacing steps and border radii.

Overriding the Global Theme

To change Vanduo's default blue to your brand's purple, simply redefine --color-primary on the :root element in your own CSS file. Ensure your CSS is loaded after Vanduo's CSS.

:root {
  /* Change the primary brand color to Brand Purple */
  --color-primary: #8a2be2;
  
  /* Buttons and links automatically inherit this change! */
  
  /* You can also adjust foundational spacing */
  --grid-gutter: 2rem;
  --btn-border-radius: 9999px; /* Pill-shaped buttons globally */
}

How Dark Mode Works Under the Hood

Unlike frameworks that require a separate CSS file or write heavy contextual selectors like .dark .btn { ... }, Vanduo handles dark mode purely by re-assigning semantic color variables at the root level.

When the data-theme="dark" attribute is added to the <html> tag, Vanduo swaps the literal hex values assigned to structural variables like --bg-primary and --text-primary.

/* By default (Light Mode) */
:root {
  --bg-primary: #ffffff;
  --text-primary: #1e293b;
  --border-color: #e2e8f0;
}

/* When Dark Mode is toggled on */
html[data-theme="dark"] {
  --bg-primary: #0f172a;       /* Background becomes very dark blue */
  --text-primary: #f8fafc;     /* Text becomes near white */
  --border-color: #334155;     /* Borders become dark */
}

/* Components just use the semantic variables. They don't care if it's light or dark! */
.vd-card {
  background-color: var(--bg-primary);
  color: var(--text-primary);
  border: 1px solid var(--border-color);
}
Because of this architecture, you can easily create custom themes (like "High Contrast" or "Sepia") by applying a new data attribute like data-theme="sepia" and overriding those same 5-6 semantic variables!

Component-Level Tokens

Beyond the global theme colors, Vanduo exposes scoped variables for individual components. These let you fine-tune a single component without affecting the rest of the system.

Component Token Examples What It Controls
Buttons --btn-height, --btn-border-radius, --btn-font-weight Size, shape, and text weight of all buttons
Cards --card-bg, --card-border-radius, --card-padding Background, corners, and internal spacing
Navbar --navbar-height, --navbar-bg Bar height and background color
Forms --input-border-radius, --input-border-color, --input-bg Input field appearance
Modal --modal-bg, --modal-backdrop, --modal-border-radius Dialog appearance and overlay 
:root {
  /* Make all cards more rounded with extra padding */
  --card-border-radius: 1rem;
  --card-padding: 2rem;

  /* Pill-shaped inputs */
  --input-border-radius: 9999px;

  /* Taller navbar */
  --navbar-height: 4.5rem;
}
How do I find all available tokens? Open your browser DevTools, select any Vanduo element, and look at the Computed tab — all CSS custom properties used by that element are listed there with their current values.

Creating a Custom Theme File

The recommended approach is to keep your overrides in a dedicated CSS file loaded after Vanduo's stylesheet. This keeps your customizations organized and easy to swap.

<!-- Vanduo first -->
<link rel="stylesheet" href="vanduo/dist/vanduo.min.css">

<!-- Your overrides second (wins due to cascade order) -->
<link rel="stylesheet" href="css/my-theme.css">
/* ── Brand Colors ──────────────────────────── */
:root {
  --color-primary: #6d28d9;
  --color-primary-hover: #5b21b6;
  --color-success: #059669;
  --color-danger: #dc2626;
}

/* ── Typography ───────────────────────────── */
:root {
  --font-family-sans: 'Inter', -apple-system, BlinkMacSystemFont, sans-serif;
  --font-family-mono: 'JetBrains Mono', monospace;
}

/* ── Component Tuning ─────────────────────── */
:root {
  --btn-border-radius: 0.5rem;
  --card-border-radius: 0.75rem;
  --input-border-radius: 0.5rem;
}

/* ── Dark Mode Adjustments ────────────────── */
html[data-theme="dark"] {
  --bg-primary: #18181b;
  --bg-secondary: #27272a;
  --border-color: #3f3f46;
}

Creating a Fully Custom Theme

You can go beyond light and dark by creating entirely new themes. Add a data-theme attribute to <html> and define the corresponding variables:

/* Sepia Theme */
html[data-theme="sepia"] {
  --bg-primary: #fdf6e3;
  --bg-secondary: #eee8d5;
  --text-primary: #657b83;
  --text-secondary: #586e75;
  --border-color: #d6cdb5;
  --color-primary: #b58900;
  --color-primary-hover: #a67c00;
}

/* High Contrast Theme */
html[data-theme="high-contrast"] {
  --bg-primary: #000000;
  --bg-secondary: #1a1a1a;
  --text-primary: #ffffff;
  --text-secondary: #e0e0e0;
  --border-color: #ffffff;
  --color-primary: #ffff00;
  --color-primary-hover: #cccc00;
}

Then toggle between themes with JavaScript:

function setTheme(name) {
  document.documentElement.setAttribute('data-theme', name);
  localStorage.setItem('vanduo-theme-preference', name);
}

// Usage
setTheme('sepia');
setTheme('high-contrast');
setTheme('dark');
setTheme('light');

Practical Override Recipes

Common real-world customizations you can copy and adapt:

Soft, Rounded UI

:root {
  --btn-border-radius: 9999px;
  --card-border-radius: 1.5rem;
  --input-border-radius: 9999px;
  --modal-border-radius: 1.5rem;
}

Corporate / Enterprise Feel

:root {
  --color-primary: #1e40af;
  --font-family-sans: 'Noto Sans', -apple-system, sans-serif;
  --btn-border-radius: 0.25rem;
  --card-border-radius: 0.375rem;
  --card-padding: 1.5rem;
}

Scoped Override (Single Page)

You can scope overrides to a specific page or section by targeting a parent class or ID instead of :root:

.landing-page {
  --color-primary: #059669;
  --card-border-radius: 2rem;
  --btn-border-radius: 9999px;
}

.dashboard {
  --color-primary: #6366f1;
  --card-padding: 1rem;
  --card-border-radius: 0.375rem;
}
