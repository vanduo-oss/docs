Production Best Practices Guide

This is the short path to a reliable Vanduo production app: pin versions, initialize the DOM you own, prefer canonical tokens, and keep form and HTML boundaries explicit.

Pin Assets

<link rel="stylesheet" href="https://cdn.jsdelivr.net/gh/vanduo-oss/framework@v1.4.0/dist/vanduo.min.css">
<script src="https://cdn.jsdelivr.net/gh/vanduo-oss/framework@v1.4.0/dist/vanduo.min.js"></script>

Use unpinned branches only while prototyping.

Scope Runtime Work

Vanduo.destroy(viewRoot);
viewRoot.innerHTML = nextHtml;
Vanduo.init(viewRoot);

This avoids unnecessary full-document scans and prevents duplicate handlers after repeated navigation.

Theme Safely

Use --vd-* semantic tokens for new themes, and load your theme after Vanduo.

:root {
  --vd-color-primary: #0f766e;
  --vd-bg-secondary: #f0fdfa;
  --vd-border-color: #99f6e4;
}

Form Boundaries

The main bundle includes native form defaults. For app embeds with an existing design system, load your app-level overrides after Vanduo or use more specific selectors for local form regions.

For new Vanduo markup, prefer the documented .vd-* form classes because they are the stable component contract.
