Lifecycle Manager Guide

The v1.4.1 lifecycle manager is the shared cleanup registry behind scoped destroy and reinit flows. Most users do not call it directly, but understanding it helps when building dynamic pages, SPAs, and custom components.

What It Tracks

Stateful components register cleanup functions against the element they initialize. Scoped runtime calls can then clean up only the affected subtree.

VanduoLifecycle.register(element, 'componentName', cleanupFns);
VanduoLifecycle.unregister(element, 'componentName');
VanduoLifecycle.destroyAll('componentName');
VanduoLifecycle.destroyAllInContainer(root, 'componentName');
VanduoLifecycle.has(element, 'componentName');
VanduoLifecycle.getAll();

Use the public Vanduo.destroy(root) and Vanduo.reinit(name, root) APIs for normal app code.

Dynamic Replacement
async function renderPanel(panel, url) {
  Vanduo.destroy(panel);
  panel.innerHTML = await fetch(url).then((res) => res.text());
  Vanduo.init(panel);
}
Targeted Reinit
const drawer = document.querySelector('#settings-drawer');
Vanduo.reinit('tabs', drawer);
Vanduo.reinit('lazyLoad', drawer);

Registry names are lowerCamelCase. Convenience globals can still expose PascalCase names for compatibility.
