Runtime Architecture Guide

Vanduo v1.4.1 centers the runtime around one public global, scoped initialization, and canonical lowerCamelCase component registry names. This lets static pages stay simple while SPAs and lazy-loaded apps can initialize only the DOM they just rendered.

Public Runtime

Vanduo.init(root);
Vanduo.initComponents(root);
Vanduo.reinit(name, root);
Vanduo.destroy(root);
Vanduo.destroyAll();
Vanduo.getComponent(name);

Pass a Document or Element to scope work. Omit root only when you intend to target the whole document.

Naming

Registry names lazyLoad, docSearch, fontSwitcher, gridLayout
Files and DOM hooks lazy-load.js, data-vd-lazy, data-vd-doc-search
Convenience globals VanduoLazyLoad, VanduoTooltips, VanduoTimeline

LazyLoad remains a compatibility alias for the 1.4.x line; new code should use lazyLoad.

Package Formats

Use the IIFE bundle for plain HTML pages, or import the ESM build through a bundler.

<link rel="stylesheet" href="dist/vanduo.min.css">
<script src="dist/vanduo.min.js"></script>
<script>Vanduo.init();</script>
import '@vanduo-oss/framework/dist/vanduo.css';
import { Vanduo } from '@vanduo-oss/framework';

Vanduo.init(document);
