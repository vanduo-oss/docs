Theme Customizer Walkthrough Guide

Vanduo's Theme Customizer is a floating panel that lets you — and your users — change the entire look and feel of a site at runtime: primary color, neutral palette, border radius, font family, and dark/light mode. This guide walks you through setting it up, reading values, reacting to changes, and building a fully branded experience.

What You Get

The Theme Customizer gives your site a floating settings panel with these controls:

18 Primary Colors

Black, red, orange, amber, yellow, lime, green, emerald, teal, cyan, sky, blue, indigo, violet, purple, fuchsia, pink, rose

6 Neutral Palettes

Charcoal, Slate, Gray, Zinc, Neutral, Stone — controls backgrounds, text, and borders

Radius, Font, Mode

Border radius presets, font family selection, and dark/light mode toggle

All changes persist in localStorage — your users' preferences survive page reloads and browser restarts.

Step 1 of 6 Add the Trigger Button

The customizer needs a button for users to open it. Place this anywhere — in your navbar, a floating corner, or a settings page:

<button class="vd-theme-customizer-trigger"
        data-theme-customizer-trigger
        aria-label="Open theme customizer">
    <i class="ph ph-paint-roller"></i>
</button>

How it works: The data-theme-customizer-trigger attribute tells Vanduo's JS to listen for clicks on this button and open the customizer panel.

Pro tip: The vd-theme-customizer-trigger class provides default floating-button styling (fixed position, bottom-right corner). You can omit this class and use your own styling if you want the trigger in a specific location.

Step 2 of 6 Include and Initialize

The Theme Customizer is part of Vanduo's JS bundle — no extra files needed. Just make sure you have:

<!-- CSS (required) -->
<link rel="stylesheet" href="path/to/vanduo.min.css">

<!-- JS (required) -->
<script src="path/to/vanduo.min.js"></script>
<script>
    Vanduo.init();  // First page boot: initialize the current document
</script>

That's it for a static page. Vanduo.init() scans the document, finds the trigger button, and sets up the customizer panel automatically. In an SPA or injected section, use Vanduo.init(root) for the newly mounted subtree instead.

Step 3 of 6 Try Each Control

Open the customizer and explore what each control does:

Control What It Changes Stored In
Primary Color Buttons, links, active states, focus rings — your brand color vanduo-primary-color
Neutral Color Backgrounds, borders, text — the canvas your content sits on vanduo-neutral-color
Border Radius How round corners are — from sharp (0) to pill-shaped vanduo-radius
Font Family The typeface used across the entire site vanduo-font-preference
Color Mode Light or dark theme vanduo-theme-preference
Why localStorage? When users pick a theme, it should stick between visits. The customizer saves each choice to localStorage under the keys above, and restores them automatically on page load.

Step 4 of 6 Read the User's Choices

Need to know what the user selected? Use the getState() API:

// Get the current customizer state
const themeCustomizer = Vanduo.getComponent('themeCustomizer');
const state = themeCustomizer.getState();

console.log(state);
// {
//   primary: 'violet',
//   neutral: 'slate',
//   radius: '0.375',
//   font: 'open-sans',
//   theme: 'dark'
// }

You can also read individual localStorage keys directly:

const primaryColor = localStorage.getItem('vanduo-primary-color');  // 'violet'
const currentTheme = localStorage.getItem('vanduo-theme-preference');  // 'dark'

Step 5 of 6 Set Values Programmatically

You can control the customizer from your own code — perfect for "preset themes" or onboarding flows:

const tc = Vanduo.getComponent('themeCustomizer');

// Set individual values
tc.applyPrimary('rose');       // Change primary color
tc.applyNeutral('zinc');       // Change neutral palette
tc.applyRadius('0.375');       // Change border radius scale
tc.applyFont('open-sans');     // Change font family
tc.applyTheme('dark');         // Switch to dark mode

// Or update multiple preferences together
tc.setPreferences({
    primary: 'rose',
    neutral: 'zinc',
    radius: '0.375',
    font: 'open-sans',
    theme: 'dark'
});

// Open/close the panel
tc.open();
tc.close();
tc.toggle();

// Reset everything to defaults
tc.reset();

Example: Preset Theme Buttons

Give users quick-pick options for common themes:

<button class="vd-btn vd-btn-secondary" data-theme-preset="corporate">Corporate</button>
<button class="vd-btn vd-btn-secondary" data-theme-preset="playful">Playful</button>
<button class="vd-btn vd-btn-secondary" data-theme-preset="minimal">Minimal</button>

<script>
const presets = {
    corporate: { primary: 'blue', neutral: 'slate', radius: '0.25', font: 'open-sans' },
    playful: { primary: 'fuchsia', neutral: 'stone', radius: '0.5', font: 'lato' },
    minimal: { primary: 'black', neutral: 'neutral', radius: '0', font: 'system' }
};

document.querySelectorAll('[data-theme-preset]').forEach((button) => {
    button.addEventListener('click', () => {
        const tc = Vanduo.getComponent('themeCustomizer');
        const preset = presets[button.dataset.themePreset];
        if (tc && preset) {
            tc.setPreferences(preset);
        }
    });
});
</script>

Step 6 of 6 Listen to Changes

React to user customizations in real time with custom events:

// Fired whenever any setting changes
document.addEventListener('theme:change', (e) => {
    console.log('Setting changed: ', e.detail);
    // e.detail = { type: 'primary-change', value: 'rose', state: {...} }
});

// Fired when the panel opens
document.addEventListener('theme:panel-open', () => {
    console.log('Customizer panel opened');
});

// Fired when the panel closes
document.addEventListener('theme:panel-close', () => {
    console.log('Customizer panel closed');
});

// Fired when user clicks "Reset"
document.addEventListener('theme:reset', () => {
    console.log('All settings reset to defaults');
});
Pro tip: Use the theme:change event to sync changes to your backend, apply analytics tracking, or update components that need to adapt to theme changes such as charts or maps.

Resetting and Defaults

Need to go back to square one?

// Reset all customizer settings to defaults
Vanduo.getComponent('themeCustomizer').reset();

// This writes the default values back to the 5
// customizer localStorage keys.
Watch out: reset() rewrites the 5 customizer localStorage keys with default values. Other localStorage data is untouched.

You can also customize the panel's appearance itself with CSS variables:

:root {
    --tc-panel-bg: var(--vd-bg-primary);
    --tc-panel-border: var(--vd-border-color);
    --tc-panel-shadow: var(--vd-shadow-lg);
    --tc-panel-width: 320px;
    --tc-panel-z-index: 9999;
}
