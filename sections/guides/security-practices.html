Security Practices Guide

Vanduo is a UI framework, not a security boundary. v1.4.1 tightens several defaults, but applications should still treat CMS, API, and user-provided strings as untrusted unless they are sanitized by the app.

Sanitized HTML

The shared sanitizer supports explicit inline-style control. Framework-owned Bubble, Tooltip, and Toast paths disable inline style unless they deliberately need it.

const clean = sanitizeHtml(input, {
  allowSvg: false,
  allowStyle: false
});

The public default remains compatibility-friendly in 1.4.x, so internal call sites opt into stricter behavior where needed.

Search Highlighting

docSearch.highlightTag is validated against a fixed allowlist: mark, span, strong, and em. Invalid tags fall back to mark.

Search.create({
  highlightTag: 'mark'
});

Lazy HTML

VanduoLazyLoad.loadSection() accepts same-origin and relative URLs only. Use it for trusted app partials, not untrusted third-party HTML.

  • Pin release assets in production, for example @v1.4.2.
  • Keep fetched partials under your own origin.
  • Sanitize CMS/user content before it reaches the partial.

Practical Checklist

  • Prefer text APIs when rendering user-provided values.
  • Use allowStyle: false unless inline style is explicitly trusted.
  • Use scoped Vanduo.destroy(root) before removing dynamic DOM.
  • Run package audits at least at a moderate threshold before release.
