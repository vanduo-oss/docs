Framework Integration Guide

Vanduo is designed to be completely framework-agnostic. Because it is built with pure CSS and Vanilla JavaScript modules, it can be dropped into any modern frontend framework (React, Vue, Nuxt, Svelte, etc.) seamlessly.

The Core Concept

When integrating Vanduo into a component-heavy framework, remember these two rules:

  1. CSS is Global: Import vanduo.css once in your global entry file (e.g., main.js, _app.js, or layout.vue).
  2. JS Needs the DOM: Vanduo's JavaScript components (like modals, dropdowns, and tabs) need elements to exist in the DOM before initialization. You must call Vanduo.init() after your framework has mounted the component to the screen.

Integration Examples

Select your framework below to see how to initialize Vanduo components when the view mounts.

In React or Next.js, use the useEffect hook to call Vanduo.init() after the component mounts.

import { useEffect } from 'react';
import 'vanduo/dist/vanduo.css';
import { Vanduo } from 'vanduo';

export default function App() {
  useEffect(() => {
    // Initialize Vanduo components once the DOM is ready
    Vanduo.init();
  }, []);

  return (
    <div className="vd-container">
      <button className="vd-btn vd-btn-primary" data-modal-target="my-modal">
        Open Modal
      </button>
      
      <div id="my-modal" className="vd-modal">
        {/* Modal Content */}
      </div>
    </div>
  );
}

In Vue 3 (Composition API) or Nuxt, use the onMounted hook.

<script setup>
import { onMounted } from 'vue';
import 'vanduo/dist/vanduo.css';
import { Vanduo } from 'vanduo';

onMounted(() => {
  // Initialize dynamic components inside this view
  Vanduo.init();
});
</script>

<template>
  <div class="vd-container">
    <!-- Your layout -->
    <button class="vd-btn" data-dropdown-trigger="menu-1">Dropdown</button>
    <div id="menu-1" class="vd-dropdown-menu">...</div>
  </div>
</template>

In Svelte, use the onMount lifecycle function.

<script>
  import { onMount } from 'svelte';
  import 'vanduo/dist/vanduo.css';
  import { Vanduo } from 'vanduo';

  onMount(() => {
    Vanduo.init();
  });
</script>

<div class="vd-container">
  <div class="vd-tabs">
     <!-- Tab markup -->
  </div>
</div>
Re-initialization in SPAs: When dealing with Client-Side Routing (like next/router or vue-router), navigating to a new page component replaces DOM elements. Ensure you call Vanduo.init() in the mounting phase of the new page component so Vanduo attaches its event listeners to the newly rendered elements!
