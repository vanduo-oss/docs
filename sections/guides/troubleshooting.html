Troubleshooting & FAQ Guide

Most Vanduo issues come down to load order, scoped initialization, token override order, or trusted-content boundaries. This v1.4.1 checklist follows the runtime architecture now used by the docs site itself.

#1 Interactive components do not respond

Likely cause: Vanduo initialized before the markup existed, or only the old DOM was initialized.

<script src="https://cdn.jsdelivr.net/gh/vanduo-oss/framework@v1.4.3/dist/vanduo.min.js"></script>
<script>Vanduo.init();</script>
const panel = document.querySelector('#settings-panel');
Vanduo.init(panel);

Static content such as grid, cards, typography, and utilities does not need JavaScript initialization.

#2 Components break after SPA navigation

Likely cause: your router removed initialized elements without giving Vanduo a chance to clean up listeners, observers, and timers.

function mountRoute(root) {
  Vanduo.init(root);
}

function unmountRoute(root) {
  Vanduo.destroy(root);
}

Repeated Vanduo.init(root) is designed to skip already-initialized elements, but explicit teardown is still the cleanest route-change habit.

#3 Theme overrides do not apply

Load your theme after Vanduo and override canonical --vd-* tokens.

<link rel="stylesheet" href="vanduo.min.css">
<link rel="stylesheet" href="brand-theme.css">

In v1.4.1, theme CSS should override canonical --vd-* tokens only. Unprefixed token aliases are no longer shipped.

#4 Forms look different than expected

The main bundle intentionally includes framework-wide defaults for common native form controls. Use .vd-input, .vd-form-group, .checkbox, .radio, and .switch when you want the documented component patterns.

Embedding Vanduo into an app with its own form reset? Load your app overrides after Vanduo, or scope your app form rules more specifically.
#5 Lazy-loaded content is not interactive

In v1.4.1, VanduoLazyLoad.loadSection() initializes only the injected container after same-origin HTML is loaded. If you inject HTML yourself, mirror that scoped pattern.

target.innerHTML = html;
Vanduo.init(target);
#6 HTML content is stripped or sanitized

Tooltip, Bubble, Toast, and Search rendering escape or sanitize user-facing HTML. Treat strings from users, APIs, or CMS fields as untrusted unless you sanitize them before passing them to Vanduo.

  • sanitizeHtml(input, { allowStyle: false }) strips inline style attributes.
  • docSearch.highlightTag only accepts mark, span, strong, or em.
  • VanduoLazyLoad.loadSection() rejects cross-origin section URLs.
