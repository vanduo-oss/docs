v1.2.8 March 14, 2026 Latest

Framework

Improvements
  • Improvement: Theme Switcher and Theme Customizer now coordinate state

    Theme changes now flow through the shared vanduo-theme-preference key, prevent circular updates, and only swap between PRIMARY_LIGHT and PRIMARY_DARK while the user is still using the default per-theme primary color.

Documentation

New Features
  • New: Separate Theme Switcher and Theme Customizer component docs

    Theme tooling is now documented as two distinct components. Theme Switcher covers lightweight system/light/dark toggles, while Theme Customizer focuses on full palette, neutral, radius, font, and mode control.

Updates
  • Update: Theme coordination guidance added to docs

    The Theme Customizer page now explains how it coexists with Theme Switcher, including shared preference storage and per-theme default primary colors.

  • Update: Visual docs refresh and version surfaces aligned to v1.2.8

    Changelog, docs landing, quick start, metadata, README, and llms context were refreshed during the v1.2.8 pre-release pass.

v1.2.4 March 4, 2026

Framework

Bug Fixes

    Framework release updated to v1.2.5. Runtime initialization logs, bundle banners, and build-info.json now resolve from the package version and stay in sync across generated artifacts.

    The default build now recreates dist/ and emits both minified and unminified JS/CSS outputs in one run. CI now validates cross-artifact version consistency and fails on drift before release asset upload.

Documentation

Bug Fixes
  • Fix: Incorrect CSS import path in Quick Start

    The Quick Start section was instructing users to import '@vanduo-oss/framework/dist/vanduo.min.css'. This subpath is not exposed by the package exports map, causing bundlers (Vite, Webpack 5, Rollup) to throw a subpath-not-defined error. Corrected to '@vanduo-oss/framework/css' — the proper exports-map entry.

  • Fix: Incorrect navbar markup in Getting Started guide

    The 7-step Getting Started guide contained Bootstrap-style navbar patterns that do not exist in Vanduo. vd-navbar-toggler → vd-navbar-toggle, data-toggle="collapse" and data-target="#mainNav" attributes removed, vd-navbar-collapse → vd-navbar-menu aria-hidden="true", and vd-container inside nav → vd-navbar-container. Fixed in both the Step 3 snippet and the Complete File reference.

Improvements
  • Improvement: Quick Start install options restructured

    The "Package Manager" card (formerly first with a Recommended badge) has been renamed to "With a Bundler (Vite)" and moved to the last position. CDN is now first and Recommended — the correct default for users who just want to drop Vanduo into a page. The bundler card now includes a warning that bare module specifiers require a build tool, and a pnpm create vite scaffold step before the import snippet.

v1.1.0 February 7, 2026 Breaking Changes

Framework

  • Breaking: CSS Class Prefixing

    All CSS classes now use the vd- prefix to prevent collisions with other frameworks. Examples: .btn → .vd-btn, .container → .vd-container, .card → .vd-card.

  • Breaking: Manual Initialization Required

    Framework no longer auto-initializes on page load. You must explicitly call Vanduo.init() after loading the script. This allows better control over initialization timing in SPAs and complex applications.

  • New: Multiple Module Formats

    Added ESM (vanduo.esm.js) and CJS (vanduo.cjs.js) build outputs alongside IIFE format. Modern bundlers can now tree-shake unused components.

  • New: Package.json Exports Map

    Added proper exports field in package.json for clean imports: import { Vanduo } from '@vanduo-oss/framework'.

  • Improvement: CSS Namespace Isolation

    The vd- prefix ensures Vanduo can be safely embedded in applications using Bootstrap, Tailwind, or other CSS frameworks without class name collisions.

No documentation changes in this release.
