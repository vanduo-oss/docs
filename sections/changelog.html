v1.3.6 April 16, 2026 Latest

Framework

Notes
  • No framework changes

    Framework bundle remains pinned at v1.3.5. Docs-only release focused on navigation speed and feel.

Documentation

New
  • New: Instant section navigation

    Section HTML is now cached in-memory (LRU + sessionStorage), prefetched on hover/touch and idle, served by a stale-while-revalidate service worker at sw.js, and swapped via document.startViewTransition when supported.

Updates
  • Update: Deep-link scroll lands reliably on far sections

    Search or deep-linking to bottom-of-sidebar sections (Glass, Morph, Ripple…) now uses an instant scrollIntoView that bypasses the global scroll-behavior: smooth, so concurrent layout shifts from sibling Vanduo.init() can no longer cancel the scroll mid-path and the URL is no longer hijacked by scroll-spy before landing.

  • Update: Instant mobile TOC feedback

    Sidebar / mobile TOC taps now immediately show the skeleton scroll loader and an .is-navigating pressed state, so there's no unresponsive gap during the fetch.

  • Update: Parallel & abortable deep-link preload

    preloadSectionsBefore now fetches up to 6 preceding sections in parallel, and handleRoute threads an AbortController so rapid clicks cancel stale fetches instead of racing to the wrong section.

  • Update: Offscreen render optimizations

    Sections use content-visibility: auto with contain-intrinsic-size, and #dynamic-content enables overflow-anchor to stabilize scroll during parallel inserts.

  • Update: Sidebar blends with dark theme

    The docs sidebar surface now follows the theme, removing the harsh contrast edge in dark mode.

  • Update: Home feature cards morph on click, glow on hover

    The nine feature cards on the home page now reveal their details via the Water Morph wave, and hover adds a subtle card-glow accent.

  • Update: Softer theme customizer defaults

    Docs-site ThemeCustomizer.DEFAULTS now use slate neutral and 0.5 radius.

  • Update: Docs meta version bumped to v1.3.6

    Framework CDN pin remains at @v1.3.5.

v1.3.5 April 15, 2026

Framework

New
  • New: Water Morph effect component

    A liquid wave content-swap effect. Add .vd-morph or data-vd-morph to any element, provide .vd-morph-current / .vd-morph-next content layers, and clicking triggers a radial wave that dissolves the old state and reveals the new one. Size variants (.vd-morph-sm, .vd-morph-lg), custom --morph-wave-color, and a programmatic VanduoMorph.morph(el) API are included. Respects prefers-reduced-motion.

Updates
  • Update: Default tooltip colors follow page theme

    Default tooltips (tooltips.css) now use a light surface on light UI and a dark surface on dark UI, instead of always using a high-contrast inverted chip. Explicit vd-tooltip-light / vd-tooltip-dark variants remain available.

  • Update: Tablet Playwright projects use correct browsers

    Chromium Tablet and WebKit Tablet projects in playwright.config.ts now set browserName explicitly (chromium / webkit) so tablet runs match their names and CI results stay reliable.

  • Update: CI tablet test job timeout increased

    The GitHub Actions tablet workflow job timeout was raised from 20 to 45 minutes so full tablet suites can finish without spurious cancellations on slower runners.

Documentation

  • New: Docs Components/Guides toggle uses framework Water Morph

    The sidebar toggle button now uses the framework .vd-morph classes instead of duplicated CSS keyframes, reducing docs-specific CSS by ~260 lines while keeping the same liquid wave animation.

Updates
  • Update: Glass docs demos now emphasize visual distinctions

    The glass documentation page (docs/sections/effects/glass.html) now includes a “Why Distinctions Can Look Subtle” explainer, higher-frequency comparison backdrops, side-by-side baseline vs glass sample rows for all size variants and modifiers, and interactive diagnostics controls for backdrop mode plus forced floating hover preview.

  • Update: E2E search tests match global search UX

    Playwright specs for home page views and search (page-views.spec.ts, search.spec.ts) now assert the global search modal and trigger flow instead of removed hero inline search inputs, keeping CI aligned with the current site.

Labs

v1.3.4 April 14, 2026

Framework

New Features
  • New: Glass Effects system — effects/glass.css

    Introduced a complete frosted-glass surface system built on backdrop-filter. Core class .vd-glass provides translucent background, specular highlight sheen, noise texture, and box shadow. Size variants .vd-glass-sm / md / lg / xl tune blur and opacity. Modifiers .vd-glass-tinted, .vd-glass-floating, and .vd-glass-contrast cover the most common use patterns. Full prefers-reduced-transparency and @supports not (backdrop-filter) fallbacks included.

  • New: Glass design tokens in tokens.css

    Added --glass-blur, --glass-saturate, --glass-bg-opacity, --glass-bg-light / dark, --glass-border-light / dark, --glass-highlight-alpha, --glass-shadow, and --glass-noise-opacity with automatic light and dark-mode overrides via [data-theme="dark"] and @media (prefers-color-scheme: dark). All tokens are aliased as --vd-glass-* in vd-aliases.css.

  • New: Glass modifier classes for existing components

    Opt-in .vd-*-glass modifier added to Navbar, Modal, Dropdown, Tooltip, Toast, Card, and FAB components, enabling frosted glass on any of them with a single class.

  • New: Scroll-activated glass — .vd-navbar-glass & data-glass-scroll

    Navbar gains scroll-aware glass activation: transparent at rest, frosted on scroll. Threshold configurable via data-scroll-threshold. A generic data-glass-scroll + data-glass-sentinel attribute API (powered by IntersectionObserver in the new glass.js component) brings the same pattern to any element. The inactive state [data-glass-scroll]:not(.is-glass-active) is CSS-only; the JS toggles .is-glass-active.

  • Update: vd-hex selection stays aligned while rotating the grid

    The labs VdHexGrid now re-syncs selectedHex references after grid regeneration, preventing stale selection coordinates during rotation, size updates, and dimension changes.

v1.3.1 March 20, 2026

Framework

Critical Security Fixes
  • Suggest — XSS in renderItems()

    renderItems() now escapes text through _escapeHtml() before applying the highlight regex via innerHTML. Previously, raw server/user data was inserted directly — a malicious suggestion could execute arbitrary scripts.

  • Select — 3 broken querySelector selectors fixed

    Selector .vd-custom-select-option corrected to .custom-select-option in updateSelectedOptions(), openDropdown(), and handleKeydown(). Keyboard navigation and programmatic updates were silently failing.

  • Select — generateId() now assigns element.id

    generateId() assigns the generated ID to element.id before returning. Previously ARIA aria-labelledby pointed to a nonexistent ID, breaking screen-reader label association.

Medium Severity Fixes
  • Code Snippet — codesnippet:copy event reports actual success

    The codesnippet:copy custom event now passes the real copySuccess result instead of a hardcoded true.

  • Modals — single shared ESC listener replaces O(n) per-modal handlers

    A single shared keydown handler on document replaces per-modal listeners. Handler is installed on first modal init and removed by destroyAll().

  • Dropdown + Select — typeahead buffer moved to per-instance state

    _typeaheadBuffer and _typeaheadTimer are now stored per-instance in the instances Map. Previously, typing in one dropdown corrupted the typeahead state of another.

  • Navbar — CSS class body-navbar-open replaces inline overflow:hidden

    Replaced document.body.style.overflow = 'hidden' with the CSS class .body-navbar-open. Prevents the navbar and modal scroll locks from clobbering each other.

  • Helpers — on() delegation returns the wrapper for off()

    on() now returns the bound wrapper function so callers can pass it to off() to remove delegation listeners.

  • Validate — 100-char limit on user regex patterns (ReDoS prevention)

    User-supplied regex patterns in the pattern rule are now capped at 100 characters to prevent catastrophic backtracking.

  • Validate — match rule uses CSS.escape() on param

    The match rule now applies CSS.escape() to the param before building the querySelector, preventing selector injection. Wrapped in try/catch for invalid selectors.

Low Severity Fixes
  • Image Box — addEventListener('load', {once:true}) replaces DOM0 onload

    Replaced this.img.onload = ... with addEventListener('load', ..., { once: true }). Handler is cleaned up in close() if the image hasn't loaded yet.

  • Theme Customizer — removed redundant second applyTheme() in reset()

    The duplicate this.applyTheme(this.DEFAULTS.THEME) call in reset() has been removed.

Documentation

New Features
  • New: Labs page with vd-hex (VdHexGrid) demo

    Labs is now available in the docs: an experimental vd-hex interactive canvas demo (selection, pan/zoom, terrain helpers, rotation). Full ESM source, modules, and README are published separately at github.com/vanduo-oss/labs (hex-grid.js, utils/hex-math.js).

v1.1.0 February 7, 2026 Breaking Changes

Framework

  • Breaking: CSS Class Prefixing

    All CSS classes now use the vd- prefix to prevent collisions with other frameworks. Examples: .btn → .vd-btn, .container → .vd-container, .card → .vd-card.

  • Breaking: Manual Initialization Required

    Framework no longer auto-initializes on page load. You must explicitly call Vanduo.init() after loading the script. This allows better control over initialization timing in SPAs and complex applications.

  • New: Multiple Module Formats

    Added ESM (vanduo.esm.js) and CJS (vanduo.cjs.js) build outputs alongside IIFE format. Modern bundlers can now tree-shake unused components.

  • New: Package.json Exports Map

    Added proper exports field in package.json for clean imports: import { Vanduo } from '@vanduo-oss/framework'.

  • Improvement: CSS Namespace Isolation

    The vd- prefix ensures Vanduo can be safely embedded in applications using Bootstrap, Tailwind, or other CSS frameworks without class name collisions.
