v1.1.7 February 18, 2026

Framework

  • Fix: Primary color not following OS dark/light switch

    When theme mode was set to "System", toggling the OS between dark and light mode changed backgrounds and text but left the primary accent color unchanged. Added a prefers-color-scheme media-query change listener in ThemeCustomizer that auto-switches the primary color between black (light) and amber (dark) when the user hasn't explicitly chosen a non-default color.

  • Fix: Default primary color was amber instead of black in light mode

    DEFAULTS.PRIMARY_LIGHT corrected from 'amber' to 'black' so first-time visitors in light mode see the intended neutral primary color.

  • Fix: Default neutral color was slate instead of neutral

    DEFAULTS.NEUTRAL corrected from 'slate' to 'neutral' to match the intended true-neutral gray scale.

Documentation

  • Fix: Getting Started "Next Steps" card navigation

    All four "Next Steps" cards in the Getting Started guide incorrectly routed to #docs/getting-started instead of their intended guide pages. Fixed data-route attributes from docs/guides/<id> to docs/<id> to match the SPA router's section ID lookup.

  • Improvement: Code examples upgraded to vd-code-snippet

    All component documentation now displays code using beautified vd-code-snippet components. Raw code-example blocks replaced with collapsible snippets featuring syntax highlighting (code-tag, code-attr, code-string), copy buttons, and language tabs (HTML, CSS, JavaScript) where applicable. Updated 27 files across Components, Data-display, Feedback, Interactive, Core, and Quick Start sections.

v1.1.0 February 7, 2026 Breaking Changes

Framework

  • Breaking: CSS Class Prefixing

    All CSS classes now use the vd- prefix to prevent collisions with other frameworks. Examples: .btn → .vd-btn, .container → .vd-container, .card → .vd-card.

  • Breaking: Manual Initialization Required

    Framework no longer auto-initializes on page load. You must explicitly call Vanduo.init() after loading the script. This allows better control over initialization timing in SPAs and complex applications.

  • New: Multiple Module Formats

    Added ESM (vanduo.esm.js) and CJS (vanduo.cjs.js) build outputs alongside IIFE format. Modern bundlers can now tree-shake unused components.

  • New: Package.json Exports Map

    Added proper exports field in package.json for clean imports: import { Vanduo } from 'vanduo-framework'.

  • Improvement: CSS Namespace Isolation

    The vd- prefix ensures Vanduo can be safely embedded in applications using Bootstrap, Tailwind, or other CSS frameworks without class name collisions.

No documentation changes in this release.
